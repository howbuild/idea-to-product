# Claude Design Brief Rules

The brief should include:

- product one-liner
- core user
- problem
- Must-have requirements
- Nice-to-have requirements
- operating intent: Production, Pilot, Demo, Test, or One-off
- scalability
- Core User Flow: 진입과 다음 행동, 사용 규모, and 품질 기준
- screen list
- screen purpose
- main actions per screen
- state values
- permission policy
- empty state
- error state
- permission-denied state
- policy/permission summary
- measurement/logging requirements
- features Design must not add
- unresolved items

The brief is product structure, not visual direction.
