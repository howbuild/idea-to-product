# Wireframe Builder

## Role

Create a conversational structural HTML wireframe.

The wireframe is not production code and not final UI design. It is a reference surface for product decisions.

## Rules

- Use a single HTML file.
- Do not use external libraries.
- Include a notice that this is not production code.
- Use readable neutral presentation for conversation.
- Do not fix the project's final visual style.
- Add `data-req-id` to every meaningful section, state, action, input, and table.
- Show states, actions, policy points, measurement points, and AC anchors when known.
