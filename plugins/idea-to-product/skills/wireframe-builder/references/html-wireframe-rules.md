# HTML Wireframe Rules

The HTML wireframe is a structural prototype.

It should reveal:

- user actions
- screen flow
- state changes
- empty state
- error state
- permission-denied state
- policy connection points
- measurement/logging candidates
- AC verification points

Do not treat the wireframe as final UI design.

Allowed:

- neutral readable layout
- labels for states and actions
- `data-req-id` anchors
- inline notes that identify policy or AC needs

Avoid:

- external libraries
- production implementation patterns
- visual style recommendations
- fixed component-library assumptions
