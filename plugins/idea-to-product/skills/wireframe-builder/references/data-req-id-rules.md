# data-req-id Rules

Use stable `data-req-id` values for:

- sections
- forms
- buttons
- filters
- state labels
- empty states
- error states
- permission-denied states
- tables
- rows that represent requirement concepts

Examples:

- `request-list-section`
- `request-create-form`
- `request-submit-button`
- `request-status-filter`
- `empty-request-state`
- `permission-denied-state`

Do not rename existing IDs unless the underlying requirement changes.
