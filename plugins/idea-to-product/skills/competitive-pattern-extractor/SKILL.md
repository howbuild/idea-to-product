# Competitive Pattern Extractor

## Role

Extract reusable product patterns from competitive or reference examples without copying them.

## Extract

- common state structure
- common permission policy
- common exception handling
- common measurement/logging pattern
- common Must-have boundary
- features often kept as Nice-to-have

## Output

- reference pattern
- source
- current product relevance
- priority fit: Green / Yellow / Red
- benefit
- caution
- recommendation
- recommendation reason
- user question
- decision-log sentence
