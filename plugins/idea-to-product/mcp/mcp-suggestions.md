# MCP Suggestions

Use MCP when the user's idea depends on existing material:

- design files
- previous PRDs
- issue lists
- internal policy documents
- support tickets
- spreadsheets
- wiki pages
- public reference research through Browser/Web when available
- analytics or DB sources for existing event usage when authorized

For every external source, record:

- source
- what was read
- what became a question candidate
- what the user confirmed
- what remains unresolved

## Reference Research MCP Candidates

- Browser/Web: 공개 레퍼런스 조사
- Notion: 기존 업무 문서나 정책문서 조회
- Figma: 기존 화면 흐름 참고
- GitHub: 기존 이슈, 기능 요청, 상태값 확인
- Analytics/DB: 실제 사용량과 이벤트 확인
- 문서 저장소: 업무 매뉴얼, 표준 양식 확인

## Rules

- Do not create or request secrets, tokens, or credentials.
- Do not use documents without access permission.
- Record sources and limitations.
- Do not copy external products.
