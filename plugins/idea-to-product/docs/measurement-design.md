# Measurement Design

Measurement design connects product behavior to evidence.

## What To Design

- key button clicks
- search and filter actions
- create, update, delete, cancel actions
- state changes
- operational review logs
- user flow item: 진입과 다음 행동, 사용 규모, or 품질 기준
- KPI links
- information not to record
- privacy and sensitive information cautions

## Useful Question

```md
이 버튼을 눌렀다는 기록을 남겨야 하나요?
```

This helps the user decide whether an action is worth tracking in the current version.

Use flow questions when useful:

```md
사용자가 어디서 들어와서 이 버튼까지 왔는지 기록해야 하나요?
이 행동이 하루에 몇 번 일어나는지 봐야 하나요?
사용자가 많아졌을 때 실패나 잘못된 정보가 얼마나 생기는지 봐야 하나요?
```
