# Session Demo Guide

Use this quick path for demos.

1. Start with a generic request board idea.
2. Create the Core Intent Card.
3. Apply the Core User Flow: 진입과 다음 행동, 사용 규모, and 품질 기준.
4. Generate a structural wireframe.
5. Ask a concrete question tied to `data-req-id`.
6. Add "실시간 채팅" and run feature drift review.
7. Generate PRD, AC, POLICY, MEASUREMENT, DECISION_LOG, COMPLETENESS, and REVIEW_REPORT.
8. Create Design handoff brief and Claude Design prompt.
9. Import design feedback and review scope before accepting changes.
