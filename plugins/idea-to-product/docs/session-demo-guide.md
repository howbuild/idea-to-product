# Session Demo Guide

Use this quick path for demos.

1. Start with a generic request board idea.
2. Create the Core Intent Card.
3. Generate a structural wireframe.
4. Ask a question tied to `data-req-id`.
5. Add "실시간 채팅" and run feature drift review.
6. Generate PRD, AC, POLICY, MEASUREMENT, DECISION_LOG, COMPLETENESS, and REVIEW_REPORT.
7. Create Design handoff brief and Claude Design prompt.
8. Import design feedback and review scope before accepting changes.
