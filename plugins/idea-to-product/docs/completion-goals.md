# Completion Goals

## Goal 1. Marketplace-Ready Repository

The repo has root metadata, marketplace metadata, runtime plugin metadata, and a package-level plugin structure.

## Goal 2. Workflow-Centered PM System

The top-level workflow defines idea intake, Core Intent Card, Core User Flow, domain injection, structural wireframe, screen-based questions, policy extraction, measurement design, feature review, completeness, document generation, consistency review, human writing pass, and final output save.

## Goal 3. Commands As Entrypoints

Commands cover start, feature addition, wireframe refinement, drift checking, completeness audit, document generation, writing cleanup, and domain-pack injection.

## Goal 4. Hooks As Review Gates

Hooks detect scope changes, wireframe changes, policy gaps, measurement gaps, completeness gaps, document inconsistency, AI-like writing, domain-pack issues, and missing output files.

## Goal 5. Reviewer Agents

Agents cover PM, scope, product flow, policy, measurement, QA, tech, and human writing.

## Goal 6. Supporting Skills

Skills are small support units for questions, wireframes, domain context, policy extraction, measurement design, PRD writing, AC writing, decision logs, and human writing audit.

Questions must be easy to understand while preserving useful product terms such as PRD, AC, KPI, 권한정책, 상태값, 이벤트, and 로그.

## Goal 7. Domain-Neutral Core

The core repo includes a domain template and registry example only.

## Goal 8. Generic Example Output

The example uses a generic request management board and avoids industry-specific domain knowledge.

## Goal 9. Self-Check Evidence

A verification report records required file existence, design-rule checks, and intentionally unconnected runtime pieces.

## Goal 10. Reference Research Support

The plugin supports reference discovery, competitive pattern review, source quality review, and copy-risk review as support layers for better product decisions.

## Goal 11. Multi-Runtime Support

Claude Code, Codex, Claude Cowork + Claude Design, Windows, and macOS are all supported. Runtime limitations are documented instead of removing support.

## Goal 12. Claude Design Handoff

The plugin can create a Design handoff brief, Claude Design prompt, and feedback import loop without expanding product scope, losing the Core User Flow, or judging visual style.
