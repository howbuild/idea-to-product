# CLAUDE_DESIGN_PROMPT: 요청 관리 게시판

요청 관리 게시판의 제품 구조를 시각화해 주세요.

## 제품 맥락

- 핵심 사용자: 요청 등록 사용자, 요청 처리 담당자
- 문제: 요청이 흩어져 담당자 확인과 처리 상태 공유가 어렵습니다.
- Must-have 요구사항: 요청 등록, 목록 확인, 검색, 상태 필터, 상태 변경, 빈 상태, 권한 없음 상태, 상태 변경 기록
- Nice-to-have 요구사항: 실시간 채팅, 자동 배정, 복잡한 관리자 기능
- 운영 의도: Pilot
- 확장 가능성: 요청 등록, 목록 확인, 상태 변경, 기록은 확장 가능하게 둡니다. 실시간 채팅과 자동 배정은 Production 단계에서 다시 검토합니다.
- 핵심 흐름:
  - 진입과 다음 행동: 사용자는 요청을 등록한 뒤 요청 목록이나 현재 상태 확인 화면으로 이동합니다.
  - 사용 규모: 요청 등록 수, 담당자 확인 비율, 상태 변경 수를 확인합니다.
  - 품질 기준: 권한정책과 상태 변경 로그로 잘못된 변경과 민감정보 기록을 줄입니다.

## 화면에 포함할 것

- 요청 등록 영역
- 요청 목록
- 검색
- 상태 필터
- 빈 상태
- 권한 없음 상태
- 상태 변경 액션
- 상태값: 접수, 처리중, 완료, 보류

## 지켜야 할 것

- 시각 스타일은 자유롭게 제안해도 됩니다.
- 색상, 폰트, 레이아웃 취향은 강제하지 않습니다.
- 새 기능을 임의로 추가하지 마세요.
- Nice-to-have 요구사항은 명시 요청이 없으면 시안에 포함하지 마세요.
- 새 기능 아이디어가 있으면 별도 제안으로 분리해 주세요.
- 권한정책, 상태값, 측정/기록 요구사항은 유지해 주세요.
- 핵심 유저 플로우의 진입과 다음 행동, 사용 규모, 품질 기준은 유지하되 새 기능으로 확장하지 마세요.
