# CLAUDE_DESIGN_PROMPT: 요청 관리 게시판

요청 관리 게시판의 제품 구조를 시각화해 주세요.

## 제품 맥락

- 핵심 사용자: 요청 등록 사용자, 요청 처리 담당자
- 문제: 요청이 흩어져 담당자 확인과 처리 상태 공유가 어렵습니다.
- MVP 범위: 요청 등록, 목록 확인, 검색, 상태 필터, 상태 변경, 빈 상태, 권한 없음 상태, 상태 변경 기록
- 제외 범위: 실시간 채팅, 자동 배정, 복잡한 관리자 기능

## 화면에 포함할 것

- 요청 등록 영역
- 요청 목록
- 검색
- 상태 필터
- 빈 상태
- 권한 없음 상태
- 상태 변경 액션
- 상태값: 접수, 처리중, 완료, 보류

## 지켜야 할 것

- 시각 스타일은 자유롭게 제안해도 됩니다.
- 색상, 폰트, 레이아웃 취향은 강제하지 않습니다.
- 새 기능을 임의로 추가하지 마세요.
- 새 기능 아이디어가 있으면 별도 제안으로 분리해 주세요.
- 권한정책, 상태값, 측정/기록 요구사항은 유지해 주세요.
