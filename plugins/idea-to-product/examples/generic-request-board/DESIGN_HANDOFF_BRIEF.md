# DESIGN_HANDOFF_BRIEF: 요청 관리 게시판

## 제품 한 줄 설명

사용자가 요청을 등록하고 담당자가 상태를 관리하는 요청 관리 게시판입니다.

## 핵심 사용자

- 요청 등록 사용자
- 요청 처리 담당자

## 해결하려는 문제

요청이 흩어져 담당자 확인과 처리 상태 공유가 어렵습니다.

## MVP 범위

- 요청 등록
- 요청 목록 확인
- 검색
- 상태 필터
- 상태 변경
- 빈 상태
- 권한 없음 상태
- 상태 변경 기록

## 제외 범위

- 실시간 채팅
- 자동 배정
- 복잡한 관리자 기능

## 주요 화면 목록

| 화면 | 목적 | 주요 액션 | 관련 data-req-id |
|---|---|---|---|
| 요청 등록 | 사용자가 요청을 남김 | 요청 등록 | request-create-form |
| 요청 목록 | 담당자가 요청을 확인 | 검색, 필터, 상태 변경 | request-list-section |
| 빈 상태 | 요청이 없을 때 안내 | 등록 유도 | empty-request-state |
| 권한 없음 상태 | 권한 없는 상태 변경 차단 | 안내 표시 | permission-denied-state |

## 상태값

- 접수
- 처리중
- 완료
- 보류

## 권한정책

- 로그인 사용자는 요청 등록 가능
- 담당자는 상태 변경 가능

## 빈 상태

등록된 요청이 없을 때 안내를 보여줍니다.

## 오류 상태

필수값 누락 시 오류를 보여줍니다.

## 권한 없음 상태

담당자가 아닌 사용자가 상태 변경을 시도하면 권한 없음 안내를 보여줍니다.

## 주요 정책

- POL-001 요청 등록 권한
- POL-002 상태 변경 권한
- POL-003 상태 변경 기록

## 측정/기록 요구사항

- MEA-001 요청 등록 이벤트
- MEA-002 상태 변경 로그
- MEA-003 완료까지 걸린 시간

## 디자인이 임의로 추가하면 안 되는 기능

- 실시간 채팅
- 자동 배정
- 관리자 대시보드

## 미결정 사항

- 상태 변경 사유 필수 여부
- 요청 삭제 MVP 포함 여부
