# Design Feedback Loop Workflow

## Purpose

Import feedback or output from Claude Design and decide whether it changes product requirements.

## Steps

1. Read the design feedback or prototype notes.
2. Separate visual style from product behavior.
3. Extract new screens, actions, states, policies, or measurement candidates.
4. Run scope review for new features.
5. Run policy review for new policy or permission implications.
6. Run measurement review for new actions or state changes.
7. Record accepted changes in DECISION_LOG.
8. Keep rejected or deferred items in v2, excluded scope, or unresolved items.

## Rule

Idea to Product does not judge visual style. It reviews whether the design output changed product flow, state, policy, measurement, AC, or scope.
