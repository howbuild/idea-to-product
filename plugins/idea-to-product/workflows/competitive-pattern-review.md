# Competitive Pattern Review Workflow

## Role

Review whether a pattern found in a reference or competitor should apply to the current product.

## Review Criteria

- Core Intent와 연결되는가
- 핵심 사용자에게 필요한가
- Must-have에 꼭 필요한가
- 운영 복잡도를 늘리는가
- 개발 범위를 과하게 키우는가
- 정책/권한/상태 설계가 필요한가
- 측정/기록 설계가 필요한가

## Output Format

```md
참고 패턴:
출처:
현재 제품과의 관련성:
우선순위 적합도:
- Green
- Yellow
- Red

장점:
주의점:
내 추천:
추천 이유:
사용자에게 물어볼 질문:
결정근거에 남길 문장:
```

## Rule

Do not copy competitors. Distinguish reference pattern from applied requirement.
